# 剧本杀同伴数据库

AstrBot 群聊剧本杀辅助插件。v2 将旧称呼中的 “NPC / 物品栏” 改为“同伴 / 同伴栏”，并把同伴、皮肤、道具拆为独立资料，支持本地图片上传、同伴养成、皮肤、专属物品、打卡、5 连抽和道具栏。

插件 ID 保持 `astrbot_plugin_juben_npc`，因此已有 AstrBot 安装、角色、星币与经验数据可以继续使用。

## 玩家指令

```text
打卡                         # 直接触发；每天一次，随机 1–5 星币，各 20%
抽奖                         # 直接触发；消耗 10 星币，固定 5 抽
状态栏 [同伴或皮肤名称]       # 直接触发；查看当前同伴状态
同伴栏 [页码]                 # 直接触发；查看自己已获得的同伴/皮肤
道具栏 [页码]                 # 直接触发；查看自己已获得的道具及查询者 ID
/使用道具名 或 使用道具名     # 消耗一个道具，显示效果说明和剩余数量
/切换同伴 名称              # 切换同伴或装备已拥有皮肤（/切换角色 仍可兼容使用）
/中奖号码                   # 群活动随机号码
```

### 无唤醒词快捷指令

`打卡`、`抽奖`、`状态栏`、`同伴栏`、`道具栏` 是游戏的五个快捷指令。它们可以直接发送，不需要 `/`、机器人唤醒词或 @机器人；插件会在消息进入 AI 前处理并返回结果，因此不会把这些游戏操作交给 AI 猜测。例如：`状态栏 灵儿`、`同伴栏 2`、`道具栏 3` 都可以直接在群里发送。带 `/` 的写法也兼容。

只有完整的指令才会被拦截，日常聊天里的“今天打卡了”“我想抽奖”等普通句子不会被当作游戏操作。

### 使用道具

- 发送 `使用道具名` 或 `/使用道具名`，例如 `使用回复药剂`。
- 插件只会匹配道具；经验球、同伴和皮肤不能通过该指令使用。
- 成功后自动扣除 **1 个**，并展示道具名称、后台填写的效果说明和剩余数量；数量归零时会从道具栏移除。
- “效果”是给主持人/玩家查看的结算说明，不会执行任意脚本或自动改变其他玩家数据。

旧的 `/物品栏`、`/赠送角色` 等仍作为兼容别名；旧 `NPC信息` 功能已移除，资料请查看 `/状态栏` 与 `/同伴栏`。

管理员（当前群群主、群管理员或 AstrBot 管理员）可使用：

```text
/赠送星币 @群友 数量
/赠送同伴 @群友 同伴、皮肤或道具名称
/赠送专属 @群友 同伴名称
```

群权限会通过 NapCat/OneBot 实时查询；无法查询时默认拒绝修改玩家资产。

## 同伴栏、皮肤与专属物品

- `/同伴栏` 按同伴经验从高到低排序；满经验时按品质、满级时间和获得时间稳定排序。
- 每页最多显示 6 个紧凑同伴组，使用 `/同伴栏 2` 翻页；角色框、皮肤行和上下间距均已收紧，皮肤直接贴在所属同伴下方。拥有非常多皮肤时才会续页。
- 未获得同伴不会出现；若只拥有皮肤，则会显示灰色同伴锚点和亮色缩进皮肤行。
- 皮肤始终挂在所属同伴下面，不会被排到列表末尾。
- 每位同伴可在后台填写多件专属物品（每行一件）。同伴栏每行排 3 件；后台可逐件发放，未赠送时文字/边框为灰色，赠送后使用后台设定的专属物品色板。
- 皮肤可通过 `/切换同伴 皮肤名` 装备。装备后，状态栏和打卡大图都会使用该皮肤画面。

## 道具栏、经验球与奖池

后台新增条目时选择“道具”，填写图片、名称、效果、抽取权重，并勾选“加入本期奖池”。道具不再区分普通/中级/高级；所有启用道具共用一个奖池，按权重随机。道具由玩家抽到后会出现在 `/道具栏`，道具栏同样支持分页。

后台新增“经验球”时填写图片、增加经验和抽取权重，并勾选加入奖池。经验球占原有的 75% 经验球奖励位，多个启用经验球按权重随机；抽中后直接给当前同伴加经验，**不会写入玩家道具栏，也不能从后台发放为道具**。未配置自定义经验球时会自动使用旧版三种经验球，保证已有奖池概率不变。

同伴、皮肤和道具各自有“加入本期奖池”复选框；同伴池和皮肤池各只保留一个当期条目，保存新的勾选会自动取消同类旧条目；道具可勾选多个候选并按各自权重随机。皮肤必须在后台绑定一个已存在同伴。

## 抽奖规则

每次 `/抽奖` 固定消耗 **10 星币并进行 5 抽**。结果页右上角显示“抽奖进度条”，100 抽保底：第一次保底会从已勾选的同伴/皮肤池随机一种，下一次保底会优先给另一种。抽奖前会检查同伴、皮肤、道具池是否都已配置；缺少任何一类时会提示后台补齐且不会扣星币。

单抽概率按客户设定：

| 奖励 | 概率 | 效果 |
|---|---:|---|
| 星辰经验球 | 30% | 当前同伴 +10 EXP |
| 月辉经验球 | 25% | 当前同伴 +15 EXP |
| 光明经验球 | 20% | 当前同伴 +20 EXP |
| 道具 | 13% | 从已勾选的道具中按抽取权重随机一件 |
| 同伴 | 0.8% | 从已勾选的同伴池中随机 |
| 皮肤 | 1.2% | 从已勾选的皮肤池中随机 |

这些指定概率合计为 90%。剩余 10% 当前明确显示为“未命中 / 保底进度 +1”，不会暗中挪用到其他奖项；如需把这 10% 分配给某一类奖项，应在后续规则中明确填写。

重复获得同伴或皮肤会转为当前同伴 +20 EXP。

## 打卡与状态栏模板

- 每日奖励为 1、2、3、4、5 星币，各 20%。
- 已移除免费入场券与概率文字。
- 打卡模板会按“当前皮肤 → 当前同伴 → 通用模板”确定性匹配，不再从所有模板随机抽取。
- 打卡和状态栏均支持三层图片：全背景、文字/信息面板装饰层、角色框透明层；PNG 透明通道会保留。可上传整张透明排版图，也可上传对应左/右区域图层。
- 模板支持角色缩放、偏移、默认中文字体 / 微软雅黑 / 黑体 / 宋体 / 楷体，以及 `{user_name}`、`{user_id}` 打开者变量。
- 文字行有明确的“加粗”开关；模板预览和 QQ 最终发图共用同一渲染器。
- 状态栏保持左角色、右信息的原有布局，同时能为每个角色/皮肤保存专属背景、面板和边框。

模板绑定优先级固定为：**当前装备皮肤 → 当前同伴 → 通用模板**。因此切换同伴或皮肤后，打卡和状态栏会使用对应的后台模板；不存在匹配模板时才回退到通用模板。每个模板的三层图片分别是全背景、文字/信息面板装饰层、角色框透明层，推荐将透明边框保存为 PNG。

## WebUI 运营页

在 AstrBot 插件详情页打开 `manage` 页面，可以：

1. 第一板块维护同伴、皮肤、道具与经验球；皮肤选择所属同伴，道具填写效果与抽取权重，经验球填写经验与抽取权重。
2. 勾选当前奖池内容、填写同伴专属物品、编辑中文名/英文名/品质/皮肤文字。
3. 选择图片、拖放图片，或将真实图片粘贴到上传区。图片保存到插件本地 `data/npc_assets`，**不需要图床**；当桌面版 WebUI 的文件桥接失败时，插件会自动改用安全的图片数据上传，不再只显示 `Network Error`。
4. 用每个条目的画面焦点 X/Y 调整状态栏与打卡同伴框的裁切位置。`0` 靠左/上，`0.5` 居中，`1` 靠右/下。
5. 顶部色彩控制按“同伴栏 / 状态栏 / 道具栏”分类显示，选择后只调整对应色彩。
6. 第二、三板块分别维护角色绑定的打卡/状态栏模板，上传三层图片、选择模板或单行字体、编辑文字位置、颜色、字重和柔和阴影，并实时预览。
7. 第四板块可对指定群/会话和玩家发放或扣除同伴、皮肤、道具（道具支持数量）；专属物品保留独立发放。删除总库条目和模板使用网页内二次确认。

### 玩家资产控制

第四板块的“群 / 会话 Scope ID”和“用户 ID”必须与玩家实际记录对应；可从“已记录玩家”下拉框自动带入，也可手动填写。

- **发放条目**：可发放同伴、皮肤或道具。重复发放道具会累加数量；同伴和皮肤不会重复创建。
- **扣除条目**：可收回同伴、皮肤或道具；道具按填写的数量扣除。扣除当前装备皮肤时会自动卸下；扣除当前同伴时会自动切到另一名已拥有同伴，没有同伴则清空当前选择。
- **经验球**：仅用于抽奖时给当前同伴增加经验，刻意不出现在资产发放/扣除下拉框，也不会进入玩家道具栏。
- **专属物品**：仍通过独立的“发放专属物品”控件逐件赠送；收回同伴时，该同伴名下已拥有的专属物品记录会一并移除。

### 模板与色彩控制

顶部色彩控制分为“同伴栏 / 状态栏 / 道具栏”三组，先选择组别再调色，避免无关字段混在一起。打卡模板和状态栏模板各自独立保存，并都支持绑定角色或皮肤、三层图片上传、字体、文字内容/颜色/坐标/字号/加粗以及实时预览。预览图和 QQ 实际返回图使用同一套渲染逻辑，保存前可先核对文字位置和透明边框效果。

复制“图片链接文字”不是图片上传。请复制实际图片数据（剪贴板出现图像）或保存图片后选择/拖入文件；仅支持 PNG、JPG、JPEG、WebP，单张最大 15MB、4000 万像素。

## 数据兼容与文件

插件首次加载 v2 时会为旧数据创建一次本地备份：

```text
data/players.v1.bak.json
data/characters.v1.bak.json
```

旧角色 ID、星币、经验、拥有记录和当前角色会保留；旧 `skin` 文本会迁为同伴的英文副标题，不会错误地把它们授予为真实皮肤。

主要文件：

```text
data/characters.json          同伴、皮肤、道具、经验球总库
data/players.json             玩家资产、皮肤、道具、专属物品、保底进度
data/settings.json            同伴栏/状态栏/道具栏色板
data/checkin_templates.json   打卡背景及逐图文字配置
data/status_templates.json    状态栏背景及逐图文字配置
data/npc_assets/*             同伴、皮肤、道具图片
data/checkin_assets/*         打卡三层模板图片
data/status_assets/*          状态栏三层模板图片
data/rendered/*.png           回复图片缓存
```

## 依赖

需要 Pillow；中文字体会优先从 `data/fonts`、系统字体或 Noto CJK 下载位置读取。

```bash
pip install pillow
```
