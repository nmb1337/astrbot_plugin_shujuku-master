# 剧本杀同伴数据库

AstrBot 群聊剧本杀辅助插件。v2 将旧称呼中的 “NPC / 物品栏” 改为“同伴 / 同伴栏”，并把同伴、皮肤、道具拆为独立资料，支持本地图片上传、同伴养成、皮肤、专属物品、打卡、每日挖矿、5 连抽和道具栏。

插件 ID 保持 `astrbot_plugin_juben_npc`，因此已有 AstrBot 安装、角色、星币与经验数据可以继续使用。

## 玩家指令

```text
打卡                         # 直接触发；每天一次，随机 1–5 星币，各 20%
抽奖                         # 直接触发；消耗 10 星币，固定 5 抽
挖矿                         # 直接触发；拥有同伴后每天一次，从挖矿池获得一道具
状态栏 [同伴或皮肤名称]       # 直接触发；查看当前同伴状态
同伴栏 [页码]                 # 直接触发；查看自己已获得的同伴/皮肤
道具栏 [页码]                 # 直接触发；查看自己已获得的道具及查询者 ID
/使用道具名 或 使用道具名     # 消耗一个普通道具，显示效果说明和剩余数量
/切换同伴 名称 或 切换同伴名称 # 切换同伴或装备已拥有皮肤（裸消息也可用）
/中奖号码                   # 群活动随机号码
```

### 无唤醒词快捷指令

`打卡`、`抽奖`、`挖矿`、`状态栏`、`同伴栏`、`道具栏`、`切换同伴` 是快捷指令。它们可以直接发送，不需要 `/`、机器人唤醒词或 @机器人；插件会在消息进入 AI 前处理并返回结果，因此不会把这些游戏操作交给 AI 猜测。例如：`状态栏 灵儿`、`同伴栏2`、`同伴栏第二页`、`道具栏 3`、`切换同伴灵儿` 都可以直接在群里发送。带 `/` 的写法也兼容。

只有完整的指令才会被拦截，日常聊天里的“今天打卡了”“我想抽奖”等普通句子不会被当作游戏操作。

### 使用普通道具

- 发送 `使用道具名` 或 `/使用道具名`，例如 `使用回复药剂`。
- 插件只会匹配普通道具；经验球、同伴和皮肤不能通过该指令使用。
- 成功后自动扣除 **1 个**，并展示道具名称、后台填写的效果说明和剩余数量；数量归零时会从道具栏移除。
- “效果”是给主持人/玩家查看的结算说明，不会执行任意脚本或自动改变其他玩家数据。

旧的 `/物品栏`、`/赠送角色` 等仍作为兼容别名；旧 `NPC信息` 功能已移除，资料请查看 `/状态栏` 与 `/同伴栏`。

仅第六板块“赠送 QQ 白名单”中填写的 QQ 可使用：

```text
/赠送星币 @群友 数量
/赠送同伴 @群友 同伴、皮肤或道具名称
/赠送专属 @群友 同伴名称
```

白名单为空时，三个聊天赠送指令全部禁用；群主、群管理员和 AstrBot 管理员也不会绕过白名单。该限制仅作用于聊天赠送指令，WebUI 的玩家资产发放仍由 AstrBot 后台登录权限保护。

## 同伴栏、皮肤与专属物品

- `/同伴栏` 按同伴经验从高到低排序；满经验时按品质、满级时间和获得时间稳定排序。
- 每页显示最多 6 个紧凑同伴组，使用 `/同伴栏2`、`/同伴栏第二页` 或 `/同伴栏 2` 翻页；拥有很多皮肤的同伴会继续分段到下一页，不会把一张图撑得无限高。专属物品三件一行，使用不透明浅色底，避免 QQ 客户端把透明标签合成为黑块。
- 未获得同伴不会出现；若只拥有皮肤，则会显示灰色同伴锚点和亮色皮肤行。
- 皮肤始终挂在所属同伴下面，不会被排到列表末尾。
- 同伴卡与皮肤卡左侧对齐并收紧纵向间距；每位同伴可在后台填写多件专属物品（每行一件），同伴栏固定三件一行。后台可逐件发放；未赠送时文字/边框为灰色，赠送后使用后台设定的专属物品色板。
- 皮肤可通过 `/切换同伴 皮肤名` 装备。装备后，状态栏和打卡大图都会使用该皮肤画面。

## 道具栏、经验球与奖池

后台新增条目时选择“道具”，填写图片、名称、品质展示标签、效果和抽取权重，并勾选“加入本期奖池”。勾选会立即保存并显示确认；品质不再划分抽奖池。道具由玩家抽到后会出现在 `/道具栏`，道具栏同样支持分页。

后台新增“经验球”时填写图片、增加经验和抽取权重，并勾选加入奖池。可新增任意多条经验球（例如月辉、光明各建一条），所有启用条目只在“经验球”概率槽内按各自权重随机；抽中后直接给当前同伴加经验，**不会写入玩家道具栏，也不能从后台发放为道具**。未配置自定义经验球时会自动使用旧版三种经验球。

同伴、皮肤和道具各自有“加入本期奖池”复选框；同伴池和皮肤池各只保留一个当期条目，保存新的勾选会自动取消同类旧条目；所有勾选道具组成一个统一奖池，按各自“抽取权重”随机。皮肤必须在后台绑定一个已存在同伴。

## 抽奖规则

每次 `/抽奖` 固定消耗 **10 星币并进行 5 抽**。新玩家不会自动获得同伴：仅在账号尚未拥有任何同伴时，首次抽奖必得已勾选的同伴，紧接着下一抽必得已勾选的皮肤；已有同伴的玩家直接按下表随机。保底累计到 100 抽时，优先发放本期尚未拥有的同伴或皮肤；两者都缺时随机一种，均已拥有时给当前同伴 +20 EXP。保底按“玩家 ID + 群/会话 ID + 奖池 ID”独立累计，修改奖池 ID 即视为新一期。抽奖前会检查同伴、皮肤和统一道具池是否都已配置；缺少任何一类时会提示后台补齐且不会扣星币。打卡和挖矿仍要求先拥有同伴。

单抽概率默认如下，可在第五板块“抽奖结果设计”分别调整经验球、道具、同伴、皮肤四项；四项总和必须为 **100%**：

| 奖励 | 概率 | 效果 |
|---|---:|---|
| 经验球 | 86% | 从全部启用经验球按“抽取权重”随机，给当前同伴加经验 |
| 道具 | 13% | 从全部已勾选道具按“抽取权重”随机一件 |
| 同伴 | 0.5% | 从已勾选的同伴池中随机 |
| 皮肤 | 0.5% | 从已勾选的皮肤池中随机 |

概率总和为 **100%**，不再产生“未命中”或空包。结果图会显示抽奖人；可在“抽奖结果设计”板块上传全背景，并分别调整经验球、道具和大奖（同伴/皮肤共用）结果卡的内部填充色，右侧保持为空供背景图自行排版。

重复获得同伴或皮肤会转为当前同伴 +20 EXP。

## 打卡与状态栏模板

- 每日奖励为 1、2、3、4、5 星币，各 20%。
- 新玩家不自动发放同伴；未拥有同伴时会提示无法打卡，需先由管理员发放或通过首抽获得同伴。
- 已移除免费入场券与概率文字。
- 打卡模板会按“当前皮肤 → 当前同伴 → 通用模板”确定性匹配，不再从所有模板随机抽取。
- 打卡和状态栏模板各只保留一张全背景图。画框、角色图与排版可直接制作进背景；不再提供面板层、角色框层、角色缩放或偏移。
- 打卡模板可填写最多 5 条随机附言；每次成功打卡随机选取一条，`{message}` 仍是一条可独立定位和排版的文字行。
- 每一条文字都可以新增或删除，并可独立设置文本、X/Y、字号、颜色、字重（常规/粗/特粗）和阴影；预览和 QQ 最终发图共用同一渲染器。连续编辑时只会显示最后一次请求的预览，不会再被旧预览覆盖。新增按钮不再依赖被嵌入页禁用的弹窗。
- 打卡和状态栏模板统一固定使用“圆润可爱”字体，不提供模板或单行的字体选择框；会优先使用 `data/fonts/CuteRounded*.ttf`，未放置时回退到可显示中文的楷体/思源宋体。字重仍通过粗体与描边明确呈现。
- 状态栏可使用 `{bonus}`、`{stars}`、`{level}` / `{current}` / `{need}`、`{skill_2_name}` / `{skill_2_desc}`、`{skill_3_name}` / `{skill_3_desc}`、`{skill_5_name}` / `{skill_5_desc}` 绑定同伴资料。皮肤也可填写经验加成和同类技能资料，装备后优先显示皮肤填写值；经验条绑定当前同伴经验，可启用/隐藏、通过 X/Y/宽高滑杆和数字输入调整位置与尺寸，并调整底色和填充色。

模板绑定优先级固定为：**当前装备皮肤 → 当前同伴 → 通用模板**。因此切换同伴或皮肤后，打卡和状态栏会使用对应的后台模板；不存在匹配模板时才回退到通用模板。背景上传后直接写入当前模板并刷新预览；左侧保留为更长的可滚动模板列表，便于管理大量角色模板。

## WebUI 运营页

在 AstrBot 插件详情页打开 `manage` 页面，可以：

1. 第一板块维护同伴、皮肤、道具与经验球；皮肤选择所属同伴，道具填写展示品质、效果和抽取权重，经验球填写经验与抽取权重。
2. 勾选当前奖池内容、填写同伴专属物品、编辑中文名/英文名/品质/皮肤文字。
3. 选择图片、拖放图片，或将真实图片粘贴到上传区。图片保存到插件本地 `data/npc_assets`，**不需要图床**；当桌面版 WebUI 的文件桥接失败时，插件会自动改用安全的图片数据上传，不再只显示 `Network Error`。
4. 用每个条目的画面焦点 X/Y 调整同伴栏、道具栏等缩略图的裁切位置。`0` 靠左/上，`0.5` 居中，`1` 靠右/下。
5. 顶部色彩控制按“同伴栏 / 道具栏”分类显示；状态栏文字颜色直接在各文字行中调整。
6. 第二、三板块分别维护角色绑定的打卡/状态栏模板：上传一张全背景后直接写入当前模板并刷新预览，左侧模板列表会占满编辑区高度并可滚动，文件夹标签可用于快速筛选。模板文字固定使用圆润可爱字体；可新增或删除文字行，编辑文字位置、字号、颜色、字重和阴影，并实时预览。X/Y/字号数值框加宽以便查看小数坐标；打卡可配置最多 5 条随机附言；状态栏可恢复任意预设绑定字段，文字可绑定同伴或皮肤的经验加成、星级和 2/3/5 星技能，并在同一区域通过滑杆调整绑定经验条。
7. 第四板块维护角色/皮肤绑定的挖矿模板：每个模板可上传 1–6 张随机背景；第一板块的道具可单独勾选进入挖矿池，并设置独立挖矿权重。结果图仅叠加道具图片、名称和效果，其他画面由背景图完成。
8. 第五板块维护抽奖结果设计：上传全背景，填写本期奖池 ID，调整经验球、道具、同伴、皮肤四项抽取概率（总和 100%），以及经验球、道具和大奖三类结果卡的内部填充色；不再显示同伴/皮肤奖池框。
9. 第六板块可维护聊天赠送 QQ 白名单，并对指定群/会话和玩家发放或扣除同伴、皮肤、道具（道具支持数量）；专属物品保留独立发放。删除总库条目和模板使用网页内二次确认。

### 玩家资产控制

第六板块的“群 / 会话 Scope ID”和“用户 ID”必须与玩家实际记录对应；可从“已记录玩家”下拉框自动带入，也可手动填写。

- **发放条目**：可发放同伴、皮肤或普通道具。重复发放普通道具会累加数量；同伴和皮肤不会重复创建。
- **扣除条目**：可收回同伴、皮肤或普通道具；道具按填写的数量扣除。扣除当前装备皮肤时会自动卸下；扣除当前同伴时会自动切到另一名已拥有同伴，没有同伴则清空当前选择。
- **经验球**：仅用于抽奖时给当前同伴增加经验，刻意不出现在资产发放/扣除下拉框，也不会进入玩家道具栏。
- **专属物品**：仍通过独立的“发放专属物品”控件逐件赠送；收回同伴时，该同伴名下已拥有的专属物品记录会一并移除。

### 模板与色彩控制

顶部色彩控制分为“同伴栏 / 道具栏”两组，先选择组别再调色，避免无关字段混在一起；状态栏文字颜色直接在各文字行中调整。总库、打卡模板、状态栏模板和挖矿模板均可填写文件夹名并按文件夹筛选。打卡模板和状态栏模板各自独立保存，并都支持绑定角色或皮肤、一张背景图、固定圆润可爱字体、文字内容/颜色/坐标/字号/字重/阴影以及实时预览；不再显示模板或单行的字体选择框。预览图和 QQ 实际返回图使用同一套渲染逻辑，保存前可先核对文字和经验条位置。

复制“图片链接文字”不是图片上传。请复制实际图片数据（剪贴板出现图像）或保存图片后选择/拖入文件；仅支持 PNG、JPG、JPEG、WebP，单张最大 15MB、4000 万像素。

## 数据兼容与文件

插件首次加载 v2 时会为旧数据创建一次本地备份：

```text
data/players.v1.bak.json
data/characters.v1.bak.json
```

旧角色 ID、星币、经验、拥有记录和当前角色会保留；旧 `skin` 文本会迁为同伴的英文副标题，不会错误地把它们授予为真实皮肤。

主要文件：

```text
data/characters.json          同伴、皮肤、道具、经验球总库
data/players.json             按 QQ 用户共享的玩家资产、皮肤、道具、专属物品；保底按群/会话与奖池 ID 分开保存
data/settings.json            同伴栏/状态栏/道具栏色板
data/gift_operators.json      可使用聊天赠送指令的 QQ 白名单
data/checkin_templates.json   打卡背景及逐图文字配置
data/status_templates.json    状态栏背景及逐图文字配置
data/mining_templates.json    挖矿模板及角色/皮肤绑定
data/draw_design.json         抽奖结果背景、三类卡片填充色、奖池 ID 与四项抽取概率
data/npc_assets/*             同伴、皮肤、道具图片
data/checkin_assets/*         打卡背景图片
data/status_assets/*          状态栏背景图片
data/mining_assets/*          挖矿背景图片
data/draw_assets/*            抽奖结果背景图片
data/rendered/*.png           回复图片缓存
```

## 依赖

需要 Pillow；中文字体会优先从 `data/fonts`、系统字体或 Noto CJK 下载位置读取。

```bash
pip install pillow
```
