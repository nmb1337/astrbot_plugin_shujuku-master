# 剧本杀 NPC 数据库

AstrBot 群聊剧本杀辅助插件，支持 NPC/皮肤资料库、后台运营页面、角色图片上传、星币、每日打卡、状态栏、抽奖和长条形物品栏。

## Plugin Page

插件提供后台页面：

```text
pages/manage/index.html
```

在 AstrBot WebUI 的插件详情页打开 `manage` 页面后，可以：

- 新增或编辑角色/皮肤。
- 上传角色图片，图片会保存到 `data/npc_assets`。
- 上传打卡背景，图片会保存到 `data/checkin_assets`。
- 为每一张打卡背景单独设置标题、奖励、星币、入场券和概率文案的内容、X/Y 位置、字号、颜色和是否加粗。
- 在保存前生成与群内成品共用同一套渲染器的实时预览。
- 修改稀有度、获取途径、经验加成、介绍、技能和月度大奖池候选。
- 查看已记录玩家。
- 通过页面给指定群/会话里的指定用户赠送角色或皮肤。

如果玩家从未在群里使用过插件，后台可能没有他的记录。可以让他先发一次 `/星币` 或 `/打卡`，也可以在页面里手动填写 `scope_id` 和 `user_id` 后赠送。

## 指令

```text
/剧本杀帮助
/星币
/赠送星币 @群友 20     # 管理员发放活动星币
/赠送角色 @群友 蓝时咖啡 # 管理员赠送角色
/打卡
/状态栏
/切换角色 蓝时咖啡
/切换角色蓝时咖啡
/抽奖
/抽奖 10
/中奖号码
/物品栏
/NPC信息 蓝时咖啡
```

`/赠送星币` 和 `/赠送角色` 会直接改变玩家资产，仅允许当前 QQ 群的群主、群管理员或 AstrBot 管理员使用。QQ群身份会通过 NapCat/OneBot 实时查询；查询失败时默认拒绝执行。私聊中仅 AstrBot 管理员可以使用这两条命令。后台 Plugin Page 的赠送功能仍由 AstrBot WebUI 登录权限保护。

## 默认角色

插件只预置客户提供的以下五个主题角色，并已包含对应原图：

- 蓝时咖啡 / Winter Issue
- 贵族下午茶 / Chess Strategy
- 红馆流明 / Cafe Lumiere
- 灰堡银刃 / Silver Blade
- 蓝咖啡晨光 / Cafe Morning

之前用于占位的九个生成角色已经移除。后续仍可在 Plugin Page 上传新图片和维护角色资料。

## 打卡背景与文字模板

在 Plugin Page 的“打卡背景与文字模板”区域，可以由运营者自行新增、上传、预览、保存和删除模板：

1. 点击“新增模板”，填写模板 ID 和名称。
2. 选择 PNG、JPG、JPEG 或 WebP，点击“上传背景”。
3. 分别编辑标题、奖励、星币、入场券和概率五项文字；文本留空即可隐藏该项。
4. X/Y 使用 `0-1` 的相对坐标，例如 `0.5 / 0.5` 代表图片中心附近；字号也是相对图片宽度的比例。
5. 点击“生成效果预览”确认成品，然后点击“保存模板”。

可用动态变量：`{title}`、`{reward}`、`{coins}`、`{tickets}`、`{date}`。未知变量或误写的花括号不会导致群内打卡命令崩溃。

群内打卡固定输出 `1280×720` 的 16:9 图片。16:9 原图会完整无缝铺满；A4 或其他比例的图片会按 `cover` 方式居中放大并裁掉超出的边缘，因此也不会出现白边或缝隙。每个模板保存自己的文字配置，随机抽到不同背景时会自动使用该背景绑定的位置、颜色、字号和文案。

## 物品栏

`/物品栏` 已从九宫格改成长条列表。每个角色或皮肤占一条，后续新增很多皮肤也能继续展示。

## 打卡概率

- 1 星币：45%
- 2 星币：30%
- 3 星币：20%
- 免费入场券 1 张：5%

## 抽奖规则

- 每抽消耗 10 星币。
- 单抽时，如果有免费入场券，会优先消耗 1 张入场券。
- 大奖概率为 6%，从本月大奖池中抽取角色/皮肤。
- 抽到未拥有的大奖会解锁。
- 抽到已拥有的大奖会转化为 600 点该角色经验。
- 其余奖品为礼物，会自动转化为当前角色经验。

## 中奖号码

- `/中奖号码` 或 `/开奖` 会由机器人在固定范围 `1-100` 内随机生成一个号码。
- 用户发送命令时附带的数字不会参与开奖结果，避免人工指定中奖号。

## 经验等级

```text
1星进度：1000
2星进度：2000
3星进度：3000
4星进度：4000
5星进度：5000
满进度累计：15000
```

状态栏会显示当前星级、经验进度条、角色图片、角色名、皮肤名、介绍和 2 星、3 星、5 星技能。未解锁技能会灰色显示。

## 数据文件

插件会自动创建：

```text
data/characters.json       角色/皮肤配置
data/checkin_templates.json 打卡背景及逐图文字配置
data/players.json          玩家数据
data/npc_assets/*          角色图片
data/checkin_assets/*      打卡背景图片
data/rendered/*.png        回复图片缓存
data/fonts/*.otf           可选中文字体
```

## 依赖

需要 Pillow：

```bash
pip install pillow
```

## 中文字体

如果服务器没有中文字体，图片里的中文会变成小方块。插件启动时会按这个顺序找字体：

```text
data/fonts/*.otf 或 *.ttf
Linux 常见中文字体目录
Windows 常见中文字体目录
自动下载 Noto Sans CJK SC 到 data/fonts
```

如果服务器不能联网，可以手动下载字体并放到：

```text
data/fonts/NotoSansCJKsc-Regular.otf
data/fonts/NotoSansCJKsc-Bold.otf
```

字体来源：Noto CJK 官方仓库  
https://github.com/notofonts/noto-cjk
